== История создания приложения

Программа "Домашние финансы" была разработана в качестве удобного онлайн инструмента
для учета и мониторинга семейного бюджета. Попытки выяснить куда деваются деньги, заработанные
столь тяжким трудом предпринимались и ранее, но все сводилось к не очень эффектному и эффективному
ведению бухгалтерии в Excel.

В июне 2011 года мной была предпринята первая попытка реализовать удобное web-приложение домашней бухгалтерии.
Приложение было готово в сентябре 2011 года. Оно было реализовано на языке Java с использованием фреймворков
<a href="http://www.springsource.org/">Spring 3</a> и <a href="http://www.hibernate.org/">Hibernate 3</a>.
На роль основной UI библиотеки был выбран популярный javascript фреймворк <a href="http://www.sencha.com/products/extjs3"> ExtJS 3-й версии</a>.
В качестве web сервера использовался <a href="http://tomcat.apache.org/download-70.cgi">Apache Tomkat 7 версии</a>. Приложение было выложено на облачный хостинг
<a href="http://jelastic.com">Jelastic Java Host</a> и размещалось на нем вплоть до апреля 2012 года, когда кончился период бета-тестирования платформы Jelastic.

В конце апреля 2012 была начата работа над текущей второй версией программы "Домашние финансы". Теперь разработка
велась на языке <a href="http://www.ruby-lang.org/en/">Ruby версии 1.9.3</a> с использованием мощного фреймворка <a href="http://rubyonrails.org/">Rails 3-й версии</a>.
Основной функционал был готов к июню 2012 года. Тогда же и началось открытое тестирование приложения, размещенного на облачном хостинге <a href="http://www.heroku.com/">Heroku</a>. В период с июня по середину
августа 2012 года был также реализован функционал дневника питания (в настоящее время отключен), проводился рефакторинг кода,
исправлялись ошибки и улучшался дизайн. В конце августа 2012 года был полностью переработан интефейс сайта с использованием популярного
javascript и css фреймворка <a href="http://twitter.github.com/bootstrap/index.html">Twitter Bootstrap</a>.
Данный подход позволил избежать необходимости реализовывать дополнительную мобильную версию сайта для планшетов и телефонов.
